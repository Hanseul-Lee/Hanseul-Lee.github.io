20-11-20 금요일 미니프로젝트 마지막날 및 최종발표

Nov 22, 2020 in TIL

오늘 한 것

  • search 페이지 완성
    • popup 눌렀을 때 화면이 맨 위로 올라가는 버그 고치기
    • 독립적인 파일 스코프를 갖지 않는 JS의 문제점 때문에 변수명 합치고 통일하기
  • 팀원 각각 파트 연결되는 부분 통일하고 전체 페이지 git으로 합치기
    • popup.js 하나로 만들기(변수명 통일, 중복되는 선언문 지우기 등)
    • 로그아웃 시 주소창에 페이지 주소 입력해도 접근불가능하게 막기
    • localstorage에 curlog 키 추가하고 미로그인시 로그인페이지로 이동하게 하기
    • html titie 통일하기
    • 이미지 파일 통일하기
  • 발표 연습(18:00 ~ 18:40)
    • 발표 PPT 준비
    • 리허설 및 각자 질문해주기
  • 최종 발표(19:00 ~ 21:00)
    • 각자 맡은 파트 별로 발표
    • 넷플릭스와 비교되고 개선한 차이점 생각하기
    • 다른 팀 발표 듣기 및 피드백

미니프로젝트를 통해 얻고 싶은 것 (피드백)

  1. git flow를 통한 협업과정 익히기

  2. API 사용하기(영화 API)

  3. HTML, CSS 까먹은 부분들 다시 되새기기

  4. async/await을 통해 비동기 처리하는 것 익히기

  5. 협업과정에서 유연하게 갈등 관리하는 법 익히기

최종발표 피드백

  • API가 렌더링 되는 순서 기준 알아볼 것
  • 발표도 실전처럼, 기술면접처럼 진지하게 임하기
  • 구현하는 프로젝트의 의의를 포함하면 Best! -> 클론코딩에서는 실제 사이트의 부족한 부분을 이렇게 보완했다고 어필하는 게 중요함(1개라도 OK)
  • 내가 만든 페이지에 대해서는 완벽하게 대답할 수 있도록 준비하기
  • 나 뿐 아니라 다른 팀원들이 만든 페이지가 어떻게 구현했는지 대략적으로라도 알고 있을 것
  • 미완성이나 3차구현 등 아쉬운 부분 끝까지 완성하기
